    API Reference¶

    REST API documentation for the Zubbl backend.

    Base URL: https://api.zubbl.in

    Authentication¶

    All requests require a Bearer token in the Authorization header:

    Authorization: Bearer zubbl_<tenant_id>_<secret>

    Endpoints¶

    Health Check¶

    Check API status.

    GET /health

    Response¶

    {
  "status": "healthy",
  "version": "0.2.0",
  "timestamp": "2026-02-18T10:30:00Z"
}

    Ingest Trajectory¶

    Ingest a trajectory for learning.

    POST /ingest

    Request Body¶

    {
  "trajectory_id": "traj_abc123def456",
  "tenant_id": "tenant_001",
  "task_description": "Write a Python function to sort a list",
  "steps": [
    {
      "step_id": "step_1",
      "action": "analyze_requirements",
      "tool_name": null,
      "tool_input": null,
      "tool_output": null,
      "reasoning": "Need to understand sorting requirements",
      "tokens_used": 150,
      "latency_ms": 200
    },
    {
      "step_id": "step_2",
      "action": "write_code",
      "tool_name": "code_generator",
      "tool_input": {"language": "python"},
      "tool_output": "def sort_list(items): return sorted(items)",
      "tokens_used": 300,
      "latency_ms": 500
    }
  ],
  "status": "success",
  "total_tokens": 450,
  "total_latency_ms": 700,
  "metadata": {
    "agent": "my_agent",
    "version": "1.0"
  }
}

    Response¶

    {
  "trajectory_id": "traj_abc123def456",
  "status": "accepted",
  "message": "Trajectory ingested successfully and queued for learning"
}

    Status Codes¶

    Code Description
    200 Success
    401 Unauthorized
    422 Validation error
    500 Server error

    Query Policy¶

    Get policy recommendations for a task.

    GET /query

    Query Parameters¶

    Parameter Type Required Description
    task string Yes Task description
    context string No Additional context (JSON string)
    rl_mode string No RL mode: local, cloud

    Example Request¶

    GET /query?task=Write%20a%20Python%20sorting%20function&rl_mode=local

    Response (Policy Found)¶

    {
  "policy_id": "pol_coding_a1b2c3d4",
  "task_type": "coding",
  "recommended_actions": [
    "analyze_requirements",
    "write_code",
    "test_code",
    "refine_if_needed"
  ],
  "confidence_score": 0.85,
  "rl_mode": "local",
  "message": "Policy found"
}

    Response (No Policy)¶

    {
  "policy_id": null,
  "task_type": "coding",
  "recommended_actions": [],
  "confidence_score": 0.0,
  "rl_mode": "local",
  "message": "No policy found for this task type yet. Keep using Zubbl and it will learn!"
}

    Submit Feedback¶

    Submit feedback on a trajectory.

    POST /feedback

    Request Body¶

    {
  "feedback_id": "fb_xyz789",
  "trajectory_id": "traj_abc123def456",
  "rating": 0.8,
  "comment": "Good solution but could be more efficient",
  "source": "human"
}

    Response¶

    {
  "feedback_id": "fb_xyz789",
  "status": "accepted",
  "message": "Feedback recorded and queued for processing"
}

    Get Usage¶

    Get usage statistics for the current tenant.

    GET /usage

    Response¶

    {
  "tenant_id": "tenant_001",
  "period": "month",
  "trajectories_ingested": 1250,
  "queries_made": 3400,
  "rl_cycles_run": 890,
  "credits_used": 5000,
  "credits_remaining": 95000,
  "task_success_rate": 0.72,
  "avg_improvement": 0.23
}

    Get Job Status¶

    Check the status of a processing job.

    GET /job/{job_id}

    Response¶

    {
  "job_id": "traj_abc123def456",
  "status": "completed"
}

    Possible Statuses¶

    Status Description
    pending Job is queued
    processing Job is being processed
    completed Job finished successfully
    failed Job failed

    Error Responses¶

    All errors follow this format:

    {
  "error": "Error message here"
}

    Common Error Codes¶

    Code Description
    400 Bad Request — Invalid parameters
    401 Unauthorized — Missing or invalid token
    402 Payment Required — Insufficient credits
    404 Not Found — Resource doesn't exist
    429 Too Many Requests — Rate limit exceeded
    500 Internal Server Error

    Rate Limits¶

    Rate limits are enforced per tenant per minute:

    Plan Limit
    Free 100/min
    Local PRO 1,000/min
    Cloud PRO 5,000/min
    Team 10,000/min

    Rate limit headers:

    X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 950
X-RateLimit-Reset: 1708254000

    Webhooks (Coming Soon)¶

    Future support for webhooks on events: - trajectory.ingested - policy.updated - rl_cycle.completed
